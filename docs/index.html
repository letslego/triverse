Overview

Modern agents don't rely on a single model. They route work across a pool of LLMs, run multi-turn reasoning, and accumulate context that grows without bound. Four ideas address different layers of that problem.

LayerProject / paperProblem solved
Coordination protocol Trinity Who does what, when — Thinker, Worker, Verifier across turns
Runtime coordinator triverse Practical multi-turn loop + routing head + YAML agent pool
Harness / backend swap Omnigent pattern Same orchestration, different providers — no rewrite
Context compression compressionX Shrink transcripts & tool output; keep errors & retrieval paths

What is Trinity?

Trinity: An Evolved LLM Coordinator (Sakana AI, 2025) shows that a tiny coordinator (~20K trainable parameters) can orchestrate a pool of frontier LLMs and beat any single model in the pool — without merging weights or fine-tuning workers.

Core insight
  • A small language model's hidden state encodes enough context to route decisions.
  • A lightweight linear head picks (LLM, role) each turn.
  • Three roles offload skill: Thinker plans, Worker executes, Verifier accepts or revises.
  • The head is trained with sep-CMA-ES (derivative-free evolution), not RL.

How triverse works

triverse implements Trinity's protocol and decision surface (agent × role per turn) as a deployable Python library — with a transparent heuristic router instead of a CMA-ES-trained Qwen head.

Router scoring (heuristic head)

Each turn scores every (agent_id, role) pair:

  • Phase bias — Thinker early, Worker mid, Verifier late
  • Domain signals — math / code / reasoning keywords in query
  • Agent strengths — YAML-declared affinities
  • Diversity penalty — avoid repeating the same agent
from triverse import Coordinator, CoordConfig
from triverse.pool import ModelPool

result = Coordinator(ModelPool.default_demo()).run(
    "Implement binary search and verify correctness."
)
print(result.answer, result.total_turns)

Omnigent — the meta-harness pattern

Omnigent is an open-source agent framework and meta-harness: one orchestration layer over Claude Code, Codex, Cursor, Pi, and custom agents. triverse borrows its core idea — swap backends without rewriting coordination.

What Omnigent provides (conceptually)
  • Uniform agent runtime — YAML-defined agents with pluggable harness kinds
  • Multi-agent sessions — supervise several agents in one workflow
  • Policy & sandboxing — govern tool use and spend
  • Any model path — API keys, subscriptions, gateways, local inference

compressionX — context compression

Multi-turn coordination grows context fast: tool JSON, logs, file reads, prior turns. compressionX sits in the pipeline and shrinks payloads before they reach any LLM — preserving errors, structure, and a path back to originals via CXR.

How it all fits together

Trinity defines what multi-LLM coordination looks like. triverse runs it. Omnigent's harness pattern wires providers. compressionX keeps the transcript bounded.
